# Security Policy (Issue #44)

## Reporting a vulnerability

Please do **not** open public GitHub issues for suspected vulnerabilities.

Send a private report to: **security@fastlane.dev**

Include:

- Affected version/commit
- Impact assessment
- Reproduction steps / PoC
- Suggested mitigation (if any)

We will acknowledge receipt and follow up with triage status.

## Disclosure expectations

- We aim to validate credible reports quickly.
- Coordinate disclosure timing with maintainers until a fix is available.

## OSS support boundaries

- OSS users receive best-effort support via GitHub.
- No guaranteed SLA/response-time commitments for OSS channels.
- Hosted operational escalation paths are separate from OSS support.
